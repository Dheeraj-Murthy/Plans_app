# AGENTS.md — Plans App

## Workspace Layout

- **Flutter/Dart**: `plans_app/`. Always `cd plans_app` before running any Flutter command.
- **Rust core**: `plans_app/rust/`. Crate `plans_core` (`cdylib` + `staticlib`).
- **FRB bindings**: `plans_app/src/rust/` (auto-generated by `flutter_rust_bridge`). Config at `plans_app/flutter_rust_bridge.yaml`.
- **Root `CLAUDE.md`**: detailed architecture, theme tokens, future phases.

## Commands

```bash
cd plans_app
flutter pub get             # first run, after pubspec changes
flutter analyze             # lint — must stay at 0 issues
flutter test                # all tests
flutter test test/widget_test.dart   # single test file
flutter run -d macos        # run on macOS
flutter build macos --debug
flutter build web --release

./run_macos.sh              # builds Rust framework + runs Flutter on macOS
```

**Rust rebuild**: `bash rust/build_macos.sh` (produces `rust/target/release/plans_core.framework`). Run before `flutter run` if Rust code changed.
**Rust quick check**: `cargo check --manifest-path rust/Cargo.toml` — fast compile-only validation.
**FRB codegen**: run `flutter_rust_bridge` codegen when Rust `api/` types change.

## State Management (Riverpod)

- **`databaseServiceProvider`** throws `UnimplementedError` if not overridden. Created in `main()` and injected via `ProviderScope(overrides: [...])`. Never read it before the override is set.
- State flow: `sidebarSelectionProvider` + `tasksProvider` → `filteredTasksProvider` (derived, drives the task list).
- `SidebarSelection` is a **sealed class** (`ViewSelection(ViewType)` | `ProjectSelection(projectId)`). **Pattern-match it**, never string-compare. `ViewType` values: `inbox`, `today`, `completed`.

## Testing

- Use `FakeDatabaseService` (extends `DatabaseService`, synchronous in-memory). **Seed projects before use**: `db.seedProject(Project(id: 'default', name: 'Inbox', colorIndex: 0))`.
- **Provider test pattern**: `createContainer(db)` → `waitForTasks(container)` (or `waitForProjects`) → read/mutate → `container.dispose()`. The `await Future(() {}); await Future(() {});` flushes async providers.
- **Widget test pattern**: pump `ProviderScope` with override → `await Future.delayed(Duration(milliseconds: 300))` → `await tester.pump(...)`.
- `flutter analyze` is the authoritative source for errors. LSP may show stale errors for deleted files.

## Architecture Notes

- **Offline-first**: All mutations go through Rust → SQLite (`rusqlite` with `bundled` feature) via `flutter_rust_bridge` v2.12.0 FFI. No network calls.
- **IDs**: UUIDs generated in Rust (`uuid` crate v4). Never auto-increment.
- **Default projects** seeded on DB creation in `rust/src/db.rs`: `'default'` (Inbox), `'work'`, `'personal'`, `'ideas'`. `Task.projectId` defaults to `'default'`.
- **Fonts**: Inter `.ttf` bundled in `fonts/` (4 weights). **Do NOT use `google_fonts`** — macOS sandbox blocks HTTP downloads.
- **Dark theme only**: `themeMode: ThemeMode.dark`, hardcoded. Palette — bg `#151618`, surface `#1B1D21`, elevated `#23262B`, accent `#7C6DF2`.
- **SDK**: Dart `^3.11.5`. Key deps: `flutter_riverpod`, `go_router`, `flutter_rust_bridge 2.12.0`, `flutter_local_notifications`, `timezone`.
- **Feature layout**: `lib/features/tasks/` and `lib/features/projects/` each own models, providers, widgets, screens. `lib/shared/` has cross-cutting widgets, helpers, database, notifications.
- **macOS sandbox**: Rust framework loaded at `@rpath/plans_core.framework/plans_core`. `flutter build macos --debug` may fail due to Xcode deployment target conflict (not a code issue).
