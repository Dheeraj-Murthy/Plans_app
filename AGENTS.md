# AGENTS.md — Plans App

## Workspace Layout

- All Flutter/Dart code lives in `plans_app/`. **Always `cd plans_app` before running any Flutter command.**
- Rust core lives in `plans_app/rust/`. Dart bindings in `plans_app/src/rust/` (auto-generated by flutter_rust_bridge).
- Root-level `CLAUDE.md` has detailed architecture, theme tokens, and future phase plans.

## Commands

```bash
cd plans_app
flutter pub get             # first run, after pubspec changes
flutter analyze             # lint — must stay at 0 issues
flutter test                # all tests
flutter test test/widget_test.dart   # single test

./run_macos.sh              # builds Rust framework + runs Flutter on macOS
```

**Rust rebuild**: `bash rust/build_macos.sh` (produces `rust/target/release/plans_core.framework`). Run before `flutter run` if Rust code changed.

## State Management (Riverpod)

- `DatabaseService` is created in `main()` and injected via `ProviderScope(overrides: [...])`. It throws `UnimplementedError` if not overridden. Never read it before the override is set.
- State flow: `sidebarSelectionProvider` + `tasksProvider` → `filteredTasksProvider` (derived, drives the task list).
- `SidebarSelection` is a **sealed class** (`ViewSelection(ViewType)` | `ProjectSelection(projectId)`). **Pattern-match it**, never string-compare. `ViewType` values: `inbox`, `today`, `completed`.

## Testing

- Use `FakeDatabaseService` (extends `DatabaseService`, synchronous in-memory). **Seed projects before use**: `db.seedProject(Project(id: 'default', name: 'Inbox', colorIndex: 0))`.
- Test helper pattern: create `ProviderContainer` with `databaseServiceProvider.overrideWithValue(db)`, then call `c.read(provider)` followed by `await Future(() {}); await Future(() {});` to flush async providers. Always `container.dispose()`.
- Widget tests: pump `ProviderScope` with the override, then `await Future.delayed(Duration(milliseconds: 300))` + `await tester.pump(...)`.
- `flutter analyze` is the authoritative source for errors. LSP may show stale errors for deleted files.

## Architecture Notes

- **Offline-first**: All mutations go through Rust → SQLite via `flutter_rust_bridge` FFI. No network calls.
- **IDs**: UUIDs generated in Rust. Never auto-increment.
- **Default projects** seeded on DB creation in `rust/src/db.rs`: `'default'` (Inbox), `'work'`, `'personal'`, `'ideas'`. `Task.projectId` defaults to `'default'`.
- **Fonts**: Inter `.ttf` bundled in `fonts/`. **Do NOT use `google_fonts`** — macOS sandbox blocks HTTP downloads.
- **Dark theme only**. No light theme support.
